## 设计原则

### KISS (Keep It Simple, Stupid)
- 追求界面设计的极致简洁
- 拒绝不必要的复杂性
- 优先选择最直观的解决方案

### YAGNI (You Aren't Gonna Need It)
- 仅实现当前明确所需的功能
- 抵制过度设计和未来特性预留
- 删除未使用的代码和依赖

### DRY (Don't Repeat Yourself)
- 统一相似功能的实现方式
- 自动识别重复代码模式
- 促进代码复用和维护
