## 📋 Quick Flow 说明

本项目采用 **Quick Flow** 轨道，专注于快速开发和实施，同时保持必要的文档和质量标准。

### Quick Flow 特点
- 🚀 **轻量级文档**: 重点文档，避免过度文档化
- ⚡ **快速实施**: 直接从技术需求到实施
- 📊 **任务驱动**: 以任务分解为核心的执行方式
- 🔄 **迭代优化**: 持续改进和反馈循环

### 与 Method Track 的区别
| 特性 | Quick Flow | Method Track |
|------|------------|--------------|
| PRD | 简化版或基于现有文档 | 完整的PRD流程 |
| Epic | 技术导向的史诗 | 业务导向的史诗 |
| 故事 | 轻量级故事文档 | 详细故事文档 |
| 审批 | 最小化审批流程 | 正式审批流程 |
| 文档 | 必要文档 | 完整文档集 |
